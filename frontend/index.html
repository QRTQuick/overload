Documentation

Basic Usage

# Analyze a Python file
from overload import analyze

bugs = analyze("my_script.py")

# Each bug report contains:
# - type: Bug category
# - severity: low|medium|high|critical  
# - line: Line number (or None)
# - description: What's wrong
# - fix: How to fix it

Advanced Options

# Custom timeout
bugs = analyze("large_file.py", timeout=60)

# Analyze code string directly
code = "print('Hello World')"
bugs = analyze(code)

# Handle file-like objects
with open("script.py") as f:
    bugs = analyze(f)

Error Handling

from overload import analyze, OverloadError, RateLimitError

try:
    bugs = analyze("script.py")
except RateLimitError:
    print("Rate limit exceeded, try again later")
except OverloadError as e:
    print(f"Analysis failed: {e}")

Bug Severity Levels

CRITICAL Security vulnerabilities, data loss risks
HIGH Runtime errors, crashes, major bugs
MEDIUM Logic errors, performance issues
LOW Code smells, minor improvements

API Reference

analyze(code_or_file, timeout=30)

Analyze Python code for bugs and issues.

Parameters:

  • code_or_file (str | file): Python code string or file path
  • timeout (int): Request timeout in seconds (default: 30)

Returns:

List[Dict] - List of bug reports

Bug Report Structure:

{
    "type": "division_by_zero",
    "severity": "high", 
    "line": 42,
    "description": "Potential division by zero error",
    "fix": "Add check: if b != 0 before division"
}

Exceptions

  • OverloadError - Base exception for all errors
  • AnalysisError - Analysis failed (invalid code, server error)
  • RateLimitError - Too many requests, try again later
